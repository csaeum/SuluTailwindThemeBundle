# Tailwind CSS 4 Integration

The SuluTailwindThemeBundle provides a **theme bridge** that maps all CSS custom properties to Tailwind 4 `@theme` tokens. This eliminates the need to manually declare `@theme {}` blocks — the bridge provides ~180 design tokens ready to use as Tailwind utility classes.

## Quick setup

In your `assets/styles/app.css`:

```css
@import "tailwindcss";
@import "@itech-world/sulu-tailwind-theme-bundle";
@import "@itech-world/sulu-tailwind-theme-bundle/styles/tailwind-theme-bridge.css";
@source "../../vendor/itech-world/sulu-tailwind-theme-bundle/templates";
```

| Line | Purpose |
|------|---------|
| `@import "tailwindcss"` | Activates the Tailwind 4 compiler (must come first) |
| `@import "...bundle"` | Imports the base CSS: typography defaults, `.btn-*` classes, gallery, carousel, form components |
| `@import "...bridge.css"` | Registers all theme CSS custom properties as `@theme` tokens (colors, fonts, borders, buttons, menu) |
| `@source "...templates"` | Scans the bundle's Twig templates for utility classes used in menus and blocks |

That's it. Rebuild your assets and you're ready to use `bg-primary`, `text-error-500`, `font-heading`, etc.

## What the bridge provides

The bridge declares all CSS custom properties generated by the ThemeCompiler inside a `@theme {}` block, with sensible defaults that are overridden at runtime when the ThemeCompiler CSS loads.

### Color palettes

4 admin-configurable palettes, each with 11 shades (50→950):

| Palette | Base variable | Shade utilities | Default |
|---------|--------------|-----------------|---------|
| Primary | `--color-primary` | `bg-primary-50` ... `text-primary-950` | Indigo |
| Secondary | `--color-secondary` | `bg-secondary-50` ... `text-secondary-950` | Slate |
| Accent | `--color-accent` | `bg-accent-50` ... `text-accent-950` | Amber |
| Background | `--color-background` | `bg-background-50` ... `text-background-950` | Neutral |

### Static palettes (not admin-configurable)

3 semantic palettes with hardcoded values + 1 neutral palette:

| Palette | Base variable | Use case | Default |
|---------|--------------|----------|---------|
| Neutral | `--color-neutral-*` | Gray scale for UI elements | Tailwind neutral |
| Error | `--color-error-*` | Form validation, alerts | Red (#ef4444) |
| Warning | `--color-warning-*` | Warning messages, caution | Orange (#f97316) |
| Success | `--color-success-*` | Confirmations, positive states | Emerald (#10b981) |

### Semantic colors

| Variable | Utility | Description |
|----------|---------|-------------|
| `--color-text` | `text-text` | Default text color |
| `--color-link` | `text-link` | Link color |
| `--color-linkHover` | `text-linkHover` | Link hover color |
| `--color-border` | `border-border` | Default border color |

### Typography

| Variable | Tailwind utility | Description |
|----------|-----------------|-------------|
| `--font-heading` | `font-heading` | Heading font family (alias for `--font-family-heading`) |
| `--font-body` | `font-body` | Body font family (alias for `--font-family-body`) |
| `--font-accent` | `font-accent` | Accent font family (alias for `--font-family-accent`) |
| `--font-size-h1` ... `--font-size-h6` | `text-h1` ... `text-h6` | Per-heading font sizes |
| `--font-size-xs` ... `--font-size-5xl` | `text-xs` ... `text-5xl` | Font size scale |
| `--line-height-h1` ... `--line-height-base` | — | Line heights (use via `var()`) |

### Borders

| Variable | Tailwind utility | Description |
|----------|-----------------|-------------|
| `--radius-DEFAULT` | `rounded` | Default border radius (alias for `--border-radius`) |
| `--radius-image` | `rounded-image` | Image border radius (alias for `--border-imageRadius`) |
| `--radius-sm` | `rounded-sm` | Small radius |
| `--radius-lg` | `rounded-lg` | Large radius |

### Buttons & Menu

Button variables (`--btn-primary-bg`, `--btn-secondary-text`, etc.) and menu variables (`--menu-bg`, `--menu-text`, etc.) are also declared in the bridge. They don't generate Tailwind utility classes but are available as `var()` references and are overridden by the ThemeCompiler at runtime.

## Using theme utilities

With the bridge imported, use theme tokens directly as Tailwind classes:

```twig
{# Before (without bridge) — verbose arbitrary values #}
<div class="bg-[var(--color-primary-50)] text-[var(--color-text)] rounded-[var(--border-radius)]">
    <h2 class="font-[var(--font-family-heading)]">Title</h2>
</div>

{# After (with bridge) — clean utility classes #}
<div class="bg-primary-50 text-text rounded">
    <h2 class="font-heading">Title</h2>
</div>
```

### Color utilities

```twig
<div class="bg-primary text-white">Primary background</div>
<div class="bg-primary-100 text-primary-900">Light primary</div>
<div class="border border-error text-error">Error state</div>
<div class="bg-success-50 text-success-700">Success message</div>
<span class="text-warning-600">Warning text</span>
```

### Typography utilities

```twig
<h1 class="font-heading text-h1">Page title</h1>
<p class="font-body text-base">Body text</p>
<small class="text-xs text-secondary-500">Caption</small>
```

### Border utilities

```twig
<img class="rounded-image" src="..." alt="...">
<div class="rounded bg-primary-50 p-4">Card with theme radius</div>
```

## Adding custom colors

You can extend the bridge with your own colors by adding a `@theme` block **after** the bridge import:

```css
@import "tailwindcss";
@import "@itech-world/sulu-tailwind-theme-bundle";
@import "@itech-world/sulu-tailwind-theme-bundle/styles/tailwind-theme-bridge.css";
@source "../../vendor/itech-world/sulu-tailwind-theme-bundle/templates";

/* Your custom additions */
@theme {
    --color-brand: #ff6b35;
    --color-brand-light: #fff0e8;
    --color-brand-dark: #c44d1a;
}
```

This gives you `bg-brand`, `text-brand-light`, etc. alongside the theme tokens.

You can also **override** bridge defaults:

```css
@theme {
    /* Use a different error color */
    --color-error: #dc2626;
    --color-error-500: #dc2626;

    /* Add more font families */
    --font-display: "Playfair Display", serif;
}
```

## Advanced: manual setup (without bridge)

If you prefer not to use the bridge, you can declare individual `@theme` tokens manually. This is useful when you only need a subset of variables:

```css
@import "tailwindcss";
@import "@itech-world/sulu-tailwind-theme-bundle";
@source "../../vendor/itech-world/sulu-tailwind-theme-bundle/templates";

@theme {
    /* Only the colors you need */
    --color-primary: #6366f1;
    --color-primary-50: #eef2ff;
    --color-primary-500: #6366f1;
    --color-primary-900: #312e81;

    --color-secondary: #64748b;

    /* Font family aliases */
    --font-heading: var(--font-family-heading, sans-serif);
    --font-body: var(--font-family-body, sans-serif);

    /* Border radius alias */
    --radius-DEFAULT: var(--border-radius, 0.375rem);
}
```

> **Note:** With the manual approach, you must declare every variable you want to use as a Tailwind utility. The bridge saves you from this by providing all ~180 variables at once.

Without any `@theme` block at all, theme variables still work via the arbitrary value syntax:

```twig
<div class="bg-[var(--color-primary)] text-[var(--color-text)]">
    Works without bridge, but verbose
</div>
```

## Tailwind 4.x compatibility

The bridge is designed for **Tailwind CSS 4.0+** and uses the `@theme` directive introduced in v4.

| Tailwind version | Supported | Notes |
|------------------|-----------|-------|
| 4.0.x | Yes | Full support |
| 4.1.x+ | Yes | Forward compatible — no deprecated features used |
| 3.x | No | Tailwind 3 uses `tailwind.config.js`, not `@theme` |

### How it works under the hood

1. **Build time**: Tailwind reads the `@theme {}` block from the bridge and generates utility classes (`bg-primary`, `text-error-500`, etc.) with values referencing CSS custom properties
2. **Runtime**: The ThemeCompiler generates a CSS file (`/iw-theme/css/theme-{id}-{hash}.css`) with `:root` declarations that override the bridge defaults via CSS cascade
3. **Result**: Utility classes automatically reflect the admin-configured theme values

The bridge's default values ensure the site looks reasonable even before the ThemeCompiler runs (e.g., during first install or if no theme is assigned).
